¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Semejanzas: Java y JavaScript comparten una sintaxis similar basada en C, ambos utilizan programación orientada a objetos, emplean llaves {} para delimitar bloques de código y comparten palabras clave como if, else, while y for.

Diferencias: Java es compilado a bytecode y ejecutado en JVM, posee un tipado fuerte, y se utiliza principalmente para aplicaciones de servidor, móviles y empresariales. JavaScript, por su parte, es interpretado, se ejecuta en navegadores o Node.js, tiene tipado dinámico, y se usa principalmente para desarrollo web frontend, aunque también sirve para backend con Node.js.

¿Qué métodos tiene el objeto Date? (Menciona al menos 5)

El objeto Date en JavaScript incluye varios métodos útiles. El método getDate() devuelve el día del mes (1-31). getMonth() retorna el mes actual (0-11). getFullYear() proporciona el año en formato de 4 dígitos. getHours() devuelve la hora actual (0-23). setTime() permite establecer la fecha y hora basándose en milisegundos. Adicionalmente, toISOString() convierte la fecha a formato ISO 8601.

¿Qué métodos tienen los arreglos? (Menciona al menos 5)

Los arreglos en JavaScript disponen de numerosos métodos. push() añade elementos al final del array. pop() elimina y devuelve el último elemento. map() crea un nuevo array aplicando una función a cada elemento del original. filter() genera un nuevo array con los elementos que cumplen una condición específica. reduce() procesa el array para producir un único valor mediante una función acumuladora. También existe forEach(), que ejecuta una función para cada elemento del array sin crear un nuevo array.

¿Cómo se declara una variable con alcance local dentro de una función?

Para declarar una variable con alcance local dentro de una función, se utiliza la palabra clave let o const. Por ejemplo: function miFuncion() { let variableLocal = "Solo accesible dentro de miFuncion"; const otraVariableLocal = 42; }. Las variables declaradas de esta manera solo son accesibles dentro del ámbito de la función y no pueden ser utilizadas fuera de ella.

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Utilizar variables globales dentro de funciones conlleva varias implicaciones negativas. Genera mayor acoplamiento y dependencias ocultas en el código, aumentando su complejidad. Existe un riesgo constante de modificaciones accidentales desde cualquier parte del programa. Se dificultan los procesos de testing y depuración al no tener control sobre quién modifica estas variables. En aplicaciones grandes, pueden producirse colisiones de nombres. Además, resulta complicado rastrear los cambios de estado de estas variables. Finalmente, las variables globales permanecen en memoria durante toda la ejecución del programa, lo que puede aumentar innecesariamente el consumo de recursos.
