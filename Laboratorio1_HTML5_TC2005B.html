Q&A

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la red global de computadoras conectadas entre sí, mientras que la World Wide Web (WWW) es un sistema de información que usa Internet para acceder a documentos y recursos a través de enlaces y direcciones web.

¿Cuáles son las partes de un URL?

Un URL es la dirección de un recurso en la web. Sus partes principales son el protocolo (HTTP o HTTPS), el dominio (nombre del servidor donde está alojado el recurso), la ruta (ubicación específica del recurso en el servidor) y los parámetros (información adicional que puede modificar la solicitud).

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Los métodos HTTP permiten la comunicación entre el navegador y el servidor. GET solicita datos sin modificarlos, POST envía información al servidor para crear un recurso, PUT actualiza un recurso existente, PATCH aplica modificaciones parciales, DELETE elimina un recurso, HEAD es similar a GET pero sin devolver el contenido, OPTIONS muestra las opciones disponibles y TRACE se usa para diagnóstico

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Cuando envías un formulario con usuario y contraseña, se debe utilizar el método POST, ya que permite enviar datos de forma segura y sin que sean visibles en la URL.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Cuando accedes a una página web desde un navegador, se utiliza el método GET, porque solo solicita la información sin modificarla.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código de respuesta 200 OK significa que la solicitud fue exitosa y que el servidor ha entregado el recurso solicitado sin problemas. No indica ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

El error 404 Not Found significa que el recurso solicitado no existe en el servidor. Es responsabilidad del desarrollador corregirlo si el recurso debería existir.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El error 500 Internal Server Error significa que hubo un problema en el servidor al procesar la solicitud. En este caso, sí es responsabilidad del desarrollador corregir el problema para que el sitio funcione correctamente.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)?

Son etiquetas que la W3C aconseja no usar en nuevos desarrollos, aunque los navegadores aún deben ser compatibles con ellas para garantizar que los sitios web antiguos sigan funcionando correctamente. Ejemplo: acronym, applet, basefont, big, center, dir, font, entre otros.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Algunas de las principales diferencias entre HTML4 y HTML5 incluyen la compatibilidad con navegadores, la introducción de nuevas etiquetas, un soporte multimedia más avanzado, formularios optimizados y la incorporación de nuevas APIs.

¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla en HTML5 tiene los siguientes componentes estructurales: table (contenedor principal), thead (encabezado), tbody (cuerpo), tfoot (pie), tr (filas), th (celdas de encabezado), y td (celdas de datos). Para estilo, se pueden usar atributos como colspan, rowspan, y elementos como caption para títulos.

¿Cuáles son los principales controles de una forma HTML5?

Los principales controles de formulario en HTML5 incluyen: input (con tipos como text, password, email, number, date, color, range), textarea para texto multilínea, select para listas desplegables, button, checkbox, radio, y nuevos elementos como datalist para sugerencias y output para mostrar resultados.

¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

Los navegadores modernos como Chrome ya tienen excelente soporte para HTML5. El antiguo medidor html5test.com ya no funciona.

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información consiste en: Nacimiento (identificación de necesidades), Desarrollo (creación del sistema), Operación (uso y mantenimiento), Mantenimiento (actualizaciones y mejoras), y Muerte (cuando el sistema se vuelve obsoleto y necesita ser reemplazado).

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información (SDLC) incluye las siguientes fases: Planificación (definición de objetivos), Análisis (recopilación de requisitos), Diseño (arquitectura y especificaciones), Implementación (codificación), Pruebas (verificación y validación), Despliegue (instalación y configuración), y Mantenimiento (soporte continuo y mejoras).
